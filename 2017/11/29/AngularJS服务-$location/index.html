AngularJS服务-$location

发表于 2017-11-29 | 分类于 AngularJS

$location服务解析在浏览器地址栏中的URL（基于window.location）并且让URL在你的应用中可用。改变在地址栏中的URL会作用到$location服务，同样的，改变$location服务也会改变浏览器的地址栏。（可以使用$location进行重定向等操作）

打印结果：

内置方法：

  • absUrl() 获取当前完整的url路径 对应js的href
    http://localhost/$location/21.1%20$location.html#/foo?name=bunny#myhash
  • url() 获取当前url路径(当前url#后面的内容,包括参数和哈希值) 对应js的pathname
    /foo?name=bunny#myhash
  • path() 获取当前url的子路径(也就是当前url#后面的内容,不包括参数)
    /foo
  • protocol() 获取当前url的协议(比如http,https) 对应js的protocol
    http
  • hash() 获取哈希值 对应js的hash
    myhash
  • host() 获取主机名 对应js的hostname
    localhost
  • port() 获取端口号 对应js的port
    80 (这里就是wamp的默认端口号)
  • replace() 如果被调用，就会用改变后的URL直接替换浏览器中的历史记录，而不是在历史记录中新建一条信息，这样可以阻止『后退』
  • search() 获取url的搜索部分，以对象形式返回 对应js的search
    {“name”:”bunny”}
    其中，url,hash,search可以读写，其它只读

注意：当浏览器的URL改变时，不会重新加载整个页面。如果想要重新加载整个页面，需要使用window.location.href

内置事件

  • $locationChangeStart 当url开始发生改变

  • $locationChangeSuccess 当url改变完成
